# 题库创建操作说明

本文档描述如何在本项目中创建题目、版本、测试点与题解，并提供导入/导出路径。适用于后台管理员账号（admin）。

## 1. 入口与前置条件

- 已启动项目：`npm run dev`
- 使用管理员账号登录（具备 `admin` 角色）
- 后台入口：`/admin`

## 2. 页面入口与功能划分

- 题库管理：`/admin/problems`
  - 创建题目（基础信息 + 可选首个版本）
  - 查看题目列表与统计
  - 进入单题管理页
- 单题管理：`/admin/problems/[id]`
  - 创建新版本
  - 新增测试点
  - 新增题解
- 题单管理：`/admin/problem-sets`
  - 创建题单
  - 向题单添加题目
- 导入导出：`/admin/import-export`
  - JSON/CSV/ZIP 导入
  - JSON/CSV 导出
- 统计页：`/admin/stats`
  - 提交统计、热门题目等

## 3. 创建题目（基础信息）

页面：`/admin/problems`

填写字段说明：

- 标题：题目名称
- 难度：1~10（数字）
- 来源：可选
- 标签：逗号分隔
- 可见性：`public | private | hidden | contest`
- 时间限制 / 内存限制（用于首个版本）
- 题目描述/输入输出/约束/样例/备注（可选，填写后会自动创建首个版本）

操作流程：

1. 填写基础字段
2. 若填写题面内容（statement/format/constraints/samples），会自动创建 `v1`
3. 点击“创建题目”

## 4. 创建新版本

页面：`/admin/problems/[id]`

流程：

1. 填写新版本的题面内容（statement 等）
2. 设置 `timeLimitMs`、`memoryLimitMb`
3. 点击“创建版本”

说明：

- 每次创建版本，版本号自动递增
- 版本不会覆盖历史版本

## 5. 添加测试点

页面：`/admin/problems/[id]`

流程：

1. 选择目标版本
2. 输入 `input` / `output`
3. 设置分数、分组、是否样例、顺序
4. 点击“添加测试点”

说明：

- `groupId` 可用于分组计分
- `isSample` 为样例测试点
- `orderIndex` 用于固定顺序

## 6. 新增题解

页面：`/admin/problems/[id]`

流程：

1. 填写题解标题与内容
2. 选择关联版本（可选）
3. 点击“新增题解”

说明：

- 题解默认 `type=official`、`visibility=public`

## 7. 题单创建与维护

页面：`/admin/problem-sets`

创建题单：

1. 填写标题/描述/可见性
2. 点击“创建题单”

添加题目：

1. 填写题单 ID
2. 填写题目 ID 与顺序
3. 点击“添加题目”

## 8. 导入与导出

页面：`/admin/import-export`

### 8.1 JSON 导入

用于完整题库导入（题目+版本+测试点+题解）。格式示例：

```json
{
  "problems": [
    {
      "title": "Sample Problem",
      "difficulty": 3,
      "visibility": "public",
      "source": "import",
      "tags": ["array", "hashmap"],
      "versions": [
        {
          "statement": "Describe the problem here.",
          "statementMd": "## Describe the problem here.\n\nSupport **Markdown** and formula blocks.",
          "constraints": "1 <= n <= 1e5",
          "hints": "Try a hash map or prefix sum, depending on the problem.",
          "inputFormat": "n\\nnums...",
          "outputFormat": "answer",
          "samples": [{ "input": "3\\n1 2 3", "output": "6" }],
          "notes": "Any notes",
          "timeLimitMs": 1000,
          "memoryLimitMb": 256,
          "testcases": [
            { "input": "3\\n1 2 3", "output": "6", "score": 100, "isSample": true, "orderIndex": 1 }
          ]
        }
      ],
      "solutions": [
        { "title": "Official", "content": "Explain the approach", "type": "official", "visibility": "public" }
      ]
    }
  ]
}
```

说明：
- `statement` 是纯文本题面兜底字段。
- `statementMd` 可选；如果提供，会优先作为 Markdown 题面保存。
- `hints` 与 `notes` 已拆分：`hints` 用于做题提示，`notes` 用于备注/补充说明。

### 8.2 CSV 导入（基础字段）

CSV 仅导入基础信息（题目标题/难度/来源/标签）。第一行必须包含字段：

```
title,difficulty,visibility,source,tags
```

示例：

```
title,difficulty,visibility,source,tags
"Two Sum",2,public,"leetcode","array|hash"
```

### 8.3 ZIP 导入（含测试点文件）

ZIP 内必须包含：

- `manifest.json` 或 `problems.json`
- 若使用 `inputPath/outputPath`，对应文件需在 ZIP 内

manifest 示例：

```json
{
  "problems": [
    {
      "title": "Sample",
      "difficulty": 3,
      "versions": [
        {
          "statement": "desc",
          "timeLimitMs": 1000,
          "memoryLimitMb": 256,
          "testcases": [
            { "inputPath": "cases/1.in", "outputPath": "cases/1.out", "score": 100 }
          ]
        }
      ]
    }
  ]
}
```

### 8.4 导出

- 题库导出 JSON：`/api/admin/problems/export`
- 题库导出 CSV：`/api/admin/problems/export?format=csv`
- 题单导出 JSON：`/api/admin/problem-sets/export`
- 题单导出 CSV：`/api/admin/problem-sets/export?format=csv`

## 9. 自测与 Mock 数据

- 自测接口：`/api/admin/dev/self-test`
- 生成 Mock 数据：`/api/admin/dev/seed`

## 10. 常见问题

1) 创建题目后看不到？
- 确认已登录管理员账号
- 确认可见性设置为 `public`

2) 判题找不到测试点？
- 确认版本下已添加测试点
- 确认测试点 input/output 已正确保存

3) JSON 导入失败？
- 检查 JSON 语法与字段名称
- `testcases` 必须包含 input 或 inputUri，output 或 outputUri
